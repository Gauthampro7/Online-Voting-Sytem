# Online National Polling System

A fully functional, role-based electronic voting system built as a **pure Java command-line application** with a **MySQL database**. 

No web browser or frameworks are needed — just run and interact directly in the terminal! This project is 100% portable and can be run from a USB drive on any computer with Java installed.

## 🚀 How to Run (Zero Setup)

You do not need an IDE like Eclipse or IntelliJ to run this project. We have provided simple scripts that simulate a real-world deployment environment.

1. **Start the Database:** Double-click `start_db.bat`. This will launch the portable MariaDB server included in this folder. Keep that black window open.
2. **Compile the Code:** Double-click `compile.bat`. This translates the human-readable `.java` source code into machine-readable bytecode inside the `out/` folder. You only need to do this once (or whenever you edit the code).
3. **Run the App:** Double-click `run.bat`. This will launch the terminal interface for the voting system.

## 🛠️ How to Edit in Eclipse

If you want to view or edit the source code inside Eclipse, you must manually link the database driver:

1. Import this folder into Eclipse as a normal Java project.
2. Right-click the project folder in the left sidebar -> **Build Path** -> **Configure Build Path...**
3. Go to the **Libraries** tab at the top.
4. Click the **Add External JARs...** button on the right.
5. Navigate to this project folder, open the `lib` folder, and select `mysql-connector-j.jar`.
6. Click **Apply and Close**.
7. Now you can open `src/com/dsu/onlinevoting/OnlineVotingApp.java` and click **Run As -> Java Application**.

## 📂 Architecture & File Structure

Instead of using complex frameworks like Spring Boot (which split logic across 50+ files), we opted for a **Data-Centric Architecture** where everything is easy to trace. 

* **The Models (Database):** We don't use separate Java classes for users or votes. The MySQL tables act directly as our Models.
* **The View (Terminal):** `System.out.println()` acts as our interface.
* **The Controller (Java):** `OnlineVotingApp.java` handles all the logic, routing, and JDBC queries.

### What do the files do?
* `src/com/dsu/onlinevoting/OnlineVotingApp.java`: The core logic file containing 100% of the application code.
* `lib/mysql-connector-j.jar`: The JDBC Driver that translates standard Java commands into MySQL protocol.
* `mariadb/`: The portable database engine and your physical data storage.
* `out/`: The folder automatically generated by the compiler containing the `.class` bytecode.

## 🔒 Security & Features

* **Auto-Healing Database:** On the first run, the Java code connects to the MySQL server and runs `CREATE DATABASE IF NOT EXISTS online_voting`. It then generates all necessary tables automatically.
* **SHA-256 Hashing:** User passwords are mathematically scrambled using Java's `MessageDigest`. Plain-text passwords are never stored in the database.
* **Role-Based Menus:** Voters, Candidates, Field Officers, and Admins are locked into their specific permission menus using continuous `while(true)` loops, preventing unauthorized access.
* **Database Constraints:** A voter is prevented from casting multiple votes at the database level by a `UNIQUE` constraint on the `voter_id` column.
