================================================================================
ONLINE VOTING SYSTEM - PROJECT EXPLANATION
================================================================================

This document explains how the pure Java version of this project works, 
why it is built this way, and what each file does.

--------------------------------------------------------------------------------
1. WHY BATCH FILES (.bat) INSTEAD OF ECLIPSE?
--------------------------------------------------------------------------------
When you are learning Java, you usually click the green "Run" button in Eclipse. 
However, Eclipse is an IDE (Integrated Development Environment)—it is a tool 
meant for writing code, not for running finished applications in the real world.

When a user opens WhatsApp, Spotify, or a college project on a real computer, 
they don't open an IDE first. They just run an executable file.

The .bat (Batch) files we created replicate exactly how a real application runs 
on a Windows machine:
- compile.bat: This acts as our "Build" step. It manually calls the 'javac' 
  command. This tells the Java compiler to read your human-readable .java code 
  and translate it into machine-readable bytecode (.class files).
- run.bat: This manually calls the 'java' command to launch your application, 
  just like running an .exe file.

By using these batch files, your project becomes 100% portable. You can put 
this folder on a USB drive, plug it into any Windows computer that has Java 
installed, and run the project immediately. No Eclipse setup required!

--------------------------------------------------------------------------------
2. HOW TO RUN THIS IN ECLIPSE (If you want to)
--------------------------------------------------------------------------------
If you still want to run this specific folder inside Eclipse to edit the code 
or debug it, you must manually link the database driver. 

Here are the exact steps:
1. Import this folder into Eclipse as a normal Java project.
2. Right-click the project folder in the left sidebar -> Build Path -> Configure Build Path...
3. Go to the "Libraries" tab at the top.
4. Click the "Add External JARs..." button on the right.
5. Navigate to this folder, open the "lib" folder, and select "mysql-connector-j.jar".
6. Click "Apply and Close".
7. Now you can open OnlineVotingApp.java and click "Run As -> Java Application".

3. WHAT IS JDBC AND WHERE DOES IT COME IN?
--------------------------------------------------------------------------------
JDBC stands for "Java Database Connectivity". It is the bridge that allows your 
Java code to talk to the MySQL database. 

Without JDBC, Java and MySQL are like two people who speak completely different 
languages. Here is how JDBC solves that in our project:

- THE API (`java.sql.*`): At the top of our code, we import `java.sql.*`. These 
  are standard rules provided by Java (like `Connection`, `PreparedStatement`, 
  and `ResultSet`). It's like Java saying, "I know how to ask a database for 
  data, but I don't know the specific language of MySQL."
  
- THE DRIVER (`lib/mysql-connector-j.jar`): This is the actual JDBC Driver. It 
  was written by the MySQL team and acts as the translator. 
  
- THE CONNECTION: When our code runs `DriverManager.getConnection()`, JDBC 
  looks inside that `.jar` file we put in the `lib` folder. It uses the driver 
  to translate your Java commands into MySQL commands, sending them through an 
  invisible pipeline to the MariaDB server running in the background.

So, JDBC comes in exactly when we import `java.sql.*` and when we link the 
`.jar` file. It is the invisible translator making the whole system work!

4. EXPLAINING THE CLASSES, MODELS, AND ARCHITECTURE
--------------------------------------------------------------------------------
Usually, in complex Java frameworks, a project is split into dozens of files 
using the MVC (Model-View-Controller) pattern:
- Models: Java classes that hold data (e.g., User.java, Candidate.java).
- Views: HTML pages or graphical UI screens.
- Controllers: Java classes that process the business logic.

To make this project as easy as possible to read and understand, we chose a 
simplified "Data-Centric" architecture contained entirely in ONE file:

- THE MODELS: We do NOT have separate Java model classes. Creating them would 
  require hundreds of lines of extra code just to move data around. Instead, 
  our MySQL database tables (users, voter_profiles, votes) act as our Models. 
  The database is the ultimate "Source of Truth".
  
- THE VIEW: Our View is simply the Terminal/Command Prompt. We use 
  `System.out.println()` to display menus and data to the user.
  
- THE CONTROLLER: Our single file (`OnlineVotingApp.java`) contains all the 
  logic. It grabs input from the View (what the user types), talks directly to 
  the Models (the Database via JDBC), and prints the result back to the View.

We designed it this way so you can trace exactly how a vote is cast by simply 
scrolling down one file, rather than jumping between 10 different files.

5. THE FILE STRUCTURE: MANUAL vs AUTOMATIC
--------------------------------------------------------------------------------

[ CREATED MANUALLY BY US (The Source) ]
These are the files you write or provide. They are the actual core project.

- src/com/dsu/onlinevoting/OnlineVotingApp.java
  The only Java code file. It contains 100% of the application logic.
  
- lib/mysql-connector-j.jar
  The JDBC driver library that allows Java to translate commands to MySQL.
  
- mariadb/
  The portable database engine. This folder holds the actual MariaDB software 
  and your physical database storage (the data folder).
  
- compile.bat, run.bat, start_db.bat
  The batch scripts written to compile and launch the program easily without an IDE.
  
- Project_Explanation.txt
  This document you are reading now.


[ CREATED AUTOMATICALLY BY THE COMPUTER (The Build Output) ]
These files are generated by the computer. You should never edit them manually.

- out/
  This folder is generated automatically during the "Build" step.
  
- out/com/dsu/onlinevoting/OnlineVotingApp.class
  This is the compiled bytecode. When you run compile.bat, the Java compiler 
  reads your .java file and generates this .class file. This is the file that 
  the computer actually runs when you double-click run.bat. 
  
  If you delete the 'out' folder, the app breaks. But you can simply run 
  compile.bat again, and the computer will automatically recreate it!
