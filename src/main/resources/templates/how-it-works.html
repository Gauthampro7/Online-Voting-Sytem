Architecture & Implementation

A deep dive into how the Online National Polling System was built from the ground up.

01. Technology Stack

Java & Spring Boot

The backend engine. We used Spring Boot for rapid REST/MVC development, utilizing Spring Security for role-based access control.

MySQL & JDBC

For persistent storage. We interfaced with MySQL via Spring Data JPA and Hibernate, which abstracts complex JDBC queries into simple Java method calls.

Thymeleaf & CSS

Server-side rendering engine combined with a custom, premium 'Glassmorphism' Vanilla CSS design system for a stunning UI.

03. Database Connectivity (JDBC & JPA)

Rather than writing raw JDBC boilerplate code (`DriverManager.getConnection(...)`), we utilized Spring Data JPA. Here is how the application connects to MySQL under the hood:

# application.properties configuration
spring.datasource.url=jdbc:mysql://localhost:3306/online_voting
spring.datasource.username=root
spring.datasource.password=
spring.jpa.hibernate.ddl-auto=update
spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.MySQL8Dialect

How it works: Spring Boot reads this configuration at startup. It initializes a HikariCP connection pool, establishing a persistent JDBC connection to MySQL on port 3306. The `ddl-auto=update` property tells Hibernate to automatically generate our SQL tables based on our Java Entity classes.

04. Step-by-Step Implementation

Step 1: Entity Modeling

We defined the database schema using Java POJOs annotated with `@Entity`. For example, the `Vote` class links a `User` (voter) to a `User` (candidate) using `@ManyToOne` relationships. Hibernate translates these annotations into primary and foreign keys in MySQL.

Step 2: Repository Interfaces

We created interfaces extending `JpaRepository`. This provides us with instant CRUD operations (save, findById, delete) without writing a single line of SQL. For custom queries, Spring dynamically generates SQL based on method names like `findByConstituencyAndStatus`.

Step 3: Service & Transaction Logic

We implemented the `@Service` layer to hold business rules. Methods like `castVote()` are annotated with `@Transactional`. This ensures that if a database error occurs halfway through a complex operation, the entire JDBC transaction rolls back, preventing data corruption.

Step 4: Spring Security Configuration

We configured a `SecurityFilterChain` to protect routes. BCrypt is used to hash passwords before storing them in MySQL. We defined strict access rules: only `ROLE_ADMIN` can access `/admin/**`, ensuring that regular voters cannot tamper with candidate approvals.

Step 5: Controllers & Frontend

Finally, we built `@Controller` classes to handle HTTP requests. They fetch data from the Services and inject it into Thymeleaf templates. The frontend was styled entirely with Vanilla CSS using CSS variables (`--primary`, `--glass-bg`) to achieve a modern, responsive interface without relying on heavy frameworks.
